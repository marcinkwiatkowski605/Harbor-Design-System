# Storybook Project Structure — Design Spec

**Date:** 2026-06-16
**Status:** Approved

## Goal

Set up a monorepo project structure for Harbor Design System with Storybook (React) and a separate design tokens package (Style Dictionary). No example components — structure only, token and component content added later.

## Decisions

- **Framework:** React (`@storybook/react-vite`)
- **Repo structure:** Monorepo with npm workspaces
- **Themes:** Single theme (`light`) for now; `dark` added as a sibling folder when needed
- **Token tiers:** tier-1 (primitives), tier-2 (semantic), tier-3 (component) — folders only, no JSON files yet
- **Reference:** Subatomic Design Tokens course (packages/subatomic-design-tokens + packages/subatomic-components), adapted for React

## Full Structure

```
Harbor Design System/
├── package.json              ← npm workspaces root
├── package-lock.json
├── CLAUDE.md
├── README.md
├── docs/
│   └── writing-guides/
└── packages/
    ├── harbor-tokens/
    │   ├── package.json
    │   ├── config.js         ← Style Dictionary config
    │   ├── index.ts
    │   ├── core/
    │   │   └── tier-1-definitions/
    │   └── light/
    │       ├── scss/
    │       ├── tier-1-definitions/
    │       ├── tier-2-usage/
    │       ├── tier-3-components/
    │       └── build/        ← gitignored, generated by Style Dictionary
    └── harbor-storybook/
        ├── package.json
        ├── vite.config.ts
        ├── tsconfig.json
        ├── .storybook/
        │   ├── main.ts       ← react-vite framework config
        │   ├── preview.ts    ← global decorators, token import
        │   ├── themes.scss   ← imports built tokens from harbor-tokens
        │   └── components/
        │       ├── tier-1-tokens/
        │       │   └── tier-1-tokens.stories.tsx
        │       ├── tier-2-tokens/
        │       │   └── tier-2-tokens.stories.tsx
        │       └── tier-3-tokens/
        │           └── tier-3-tokens.stories.tsx
        └── src/
            └── components/   ← empty, components added when ready
```

## Root Scripts

| Script | Action |
|---|---|
| `npm start` | Start Storybook dev server |
| `npm run build:tokens` | Run Style Dictionary build |
| `npm run build:storybook` | Build static Storybook |

## What Is NOT in Scope

- Token JSON content (added when tokens are ready)
- React component implementations (added when components are ready)
- Dark theme (added as `packages/harbor-tokens/dark/` when needed)
- NPM publishing setup
- CI/CD
